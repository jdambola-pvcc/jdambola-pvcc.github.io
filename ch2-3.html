Chapter 2 Definitions

  1. Natural Language: Natural Language refers to the way humans communicate with each other using spoken or written language. It is the language that is used by humans to communicate with each other in their daily lives.
  2. Pseudocode: Pseudocode is an informal high-level description of an algorithm that uses a mixture of natural language and programming language-like syntax. It is used as a tool for planning and communication in the development of software and other systems.
  3. sequential operation (straight-line algorithm): A sequential operation or straight-line algorithm refers to a type of algorithm in which the instructions are executed in a linear sequence, one after the other. This means that the output of one operation is used as the input for the next operation.
  4. Input: Input refers to the data that is fed into a system, algorithm, or program to be processed or analyzed.
  5. Output: Output refers to the result of a system, algorithm, or program after it has processed or analyzed the input data.
  6. Conditional statement: A conditional statement is a programming construct that allows for the execution of different sets of instructions depending on whether a certain condition is true or false.
  7. Boolean condition: A Boolean condition is a statement that evaluates to either true or false. It is used in conditional statements to determine whether a certain set of instructions should be executed.
  8. Algorithmic discovery process: The algorithmic discovery process refers to the process of developing an algorithm or refining an existing algorithm through a combination of experimentation, analysis, and iteration. This process involves testing and refining the algorithm until it produces the desired result.

Chapter 3 Definitions

  1. Why algorithms should be correct: Algorithms should be correct because they are used to solve problems, and an incorrect algorithm can lead to incorrect or unpredictable results. A correct algorithm ensures that the problem is solved in a predictable and reliable manner.
  2. Why algorithms should be efficcient: Algorithms should be efficient because they are often used to solve large or complex problems, and an inefficient algorithm can take an unreasonable amount of time to solve the problem. An efficient algorithm ensures that the problem is solved in a reasonable amount of time.
  3. Program maintenance: Program maintenance refers to the process of updating and modifying a program after it has been deployed. This is necessary to fix bugs, improve performance, and add new features.
  4. Importance of algoritim ease of understanding: The ease of understanding an algorithm is important because it allows developers to quickly and accurately implement the algorithm, reducing the chance of errors or inefficiencies.
  5. Algorithmic elegance: Algorithmic elegance refers to the quality of an algorithm that is both efficient and easy to understand. An elegant algorithm is a balance between efficiency and simplicity.
  6. Sequential search algorithm: A sequential search algorithm is a simple algorithm that searches through a list or array one item at a time until it finds the item it is looking for.
  7. Order of magnitude: Order of magnitude is a measure of the size or scale of a value, typically represented as a power of 10.
  8. Selection sort: Selection sort is a simple sorting algorithm that works by repeatedly finding the minimum element from an unsorted part of the list and moving it to the beginning of the sorted part of the list.
  9. Flop (floating point operation): A flop, or floating point operation, is a mathematical operation that involves floating point numbers. It is often used as a measure of computer performance.
  10. Terraflop: A terraflop is a measure of computer performance that represents one trillion floating point operations per second.
  11. Petaflop: A petaflop is a measure of computer performance that represents one quadrillion floating point operations per second.
  12. Parallel processor: A parallel processor is a type of computer architecture that uses multiple processors to perform calculations simultaneously, increasing the overall speed and efficiency of the system.
  13. Data cleanup porblem: The data cleanup problem refers to the challenge of cleaning and preparing large datasets for analysis. This often involves removing duplicates, correcting errors, and standardizing the format of the data.
  14. Binary search algorithm: A binary search algorithm is a more efficient algorithm for searching through a sorted list or array by repeatedly dividing the search interval in half.
  15. Pattern matching: Pattern matching refers to the process of finding a specific pattern or sequence of characters within a larger string or data set. It is often used in data analysis and text processing.
