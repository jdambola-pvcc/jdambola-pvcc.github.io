  2. source code: Human-readable code written by a programmer in a programming language.
  3. procedural language: A programming language that uses a programming syntax that specifies a sequence of instructions that the computer can execute.
  4. Integrated Development Environment (IDE): A software application that provides tools for software development, such as source code editing, debugging, and testing.
  5. Software development life cycle (SDLC): A series of stages that software development teams use to develop and maintain software.
  6. Waterfall model of software development: A linear software development methodology that divides the development process into distinct phases, with each stage completed before the next phase begins.
  7. Agile model of software development: A software development methodology that focuses on iterative and incremental development, where requirements and solutions are developed in short sprints.

Briefly describe each of the following alternative programming paradigms

  1. Functional programming is a programming paradigm that emphasizes the use of pure functions and avoids shared state and mutable data. In functional programming, functions are treated as first-class citizens, meaning they can be passed as arguments to other functions or returned as values. The goal of functional programming is to create code that is easy to reason about and has few side effects.
  2. Logic programming is a programming paradigm that is based on formal logic. In logic programming, programs are written in terms of logical statements or rules, and the program's execution involves the derivation of logical conclusions based on these rules. This allows programmers to express problems in a declarative way, rather than specifying a step-by-step algorithm for solving them.
  3. Parallel programming is a programming paradigm that is designed to take advantage of parallel computing architectures, where multiple processors or cores can execute instructions simultaneously. Parallel programming involves dividing a problem into smaller sub-problems that can be solved concurrently, and then combining the results to produce a final solution. This can lead to significant performance improvements over sequential programming on single-core machines.
